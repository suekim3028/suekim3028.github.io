프로그레시브 웹앱이란?​

  • 기존의 모바일 앱만이 가능했던 기능들 (platform dependent)
    • 위치정보, 푸쉬 등
  • -> 이러한 기능들을 '웹 브라우저'가 지원하기 시작하면서
    • 앱 설치 없이
    • 단일 코드 베이스로
    • 모든 웹 사용자가 사용가능 해짐!
  • mdn 예시: JS 13K GAMES https://mdn.github.io/pwa-examples/js13kpwa/

프로그레시브 웹앱의 주요 기능들​

  1. Capable

    • 기존의 platform specific app 에서만 가능했던 기능들을 사용자 권한 허용을 기반으로 제공함 이미지 출처: https://medium.com/watcha/%EB%B8%8C%EB%9D%BC%EC%9A%B0%EC%A0%80%EC%97%90%EC%84%9C-%EB%AF%B8%EB%94%94%EC%96%B4-%EA%B6%8C%ED%95%9C%EC%9D%84-%EB%8B%A4%EB%A3%A8%EB%8A%94-%EA%B0%84%EB%8B%A8-tip-ee9e42ce9cb1

  2. Reliable

    • 네트워크와 관계 없이 더 빠른 환경을 제공하는 웹앱
    • 필요한 정보의 경우 잘게 나누어 다른 문제가 발생하더라도 최대한 보여줄 수 있도록
    • 에러가 발생하더라도 crash가 아닌 적절한 알림 제공
    • https에서만 사용 가능하고 제공되는 기능들은 모두 보안 기준을 가지고 있음

  3. Installable

    • 설치가 가능하다!
    • 브라우저 내에서만 동작하는 것이 아니라, 따로 standalone한 윈도우로 보여줄 수 있음
      • 유저에게 더 상호작용한 느낌을 줌

웹 앱을 설치가능하도록 만들기​

  • 가능한 방법
    1. 브라우저에서 설치가능하도록 만들기
      • manifest.json 파일을 통해 브라우저에게 이 웹앱이 어떤 방식으로 동작할지 알려줌
    2. 안드로이드에서만 가능
      • 안드로이드 앱에서, 같은 개발자가 만든 pwa를 브라우저에서 보는 것처럼 열 수 있음

PWA로 개발자가 가질 수 있는 이점​

  1. 내 앱이 검색 가능해진다!
    • 기존에 앱스토어나 플레이스토어에만 올라갔던 앱과 다르게, SEO를 통해 브라우저 검색엔진에서 검색이 가능하고 더 많은 사람에게 노출이 될 수 있음
  2. 디자인 구현에서의 제약이 낮음
  3. 앱 심사 과정이 필요 없음
  4. 일렉트론 등 데스크탑 앱 대신 PWA를 사용하는 결정을 내릴 수 있음

가능한 기능들​

  • https://developer.mozilla.org/ko/docs/Web/API 웹 브라우저에서 제공하는 다양한 API 들을 확인할 수 있음

    Web API 브라우저에서 제공하는 다양한 기능들의 Interfacee들

  1. Push API
    • 기존의 Notification API와 다르게, 브라우저가 닫힌 뒤에도 유저에게 알림을 줄 수 있음!
      • 브라우저의 Service Worker를 등록하면, 웹앱과 브라우저, 네트워크 사이에서 proxy 역할을 함 이미지 출처: https://www.telerik.com/blogs/implementing-web-push-notifications-mern-stack-application
    • service worker
      • 백그라운드에서 실행되어 네트워크 요청을 가로채고 데이터를 캐시하며 푸시 알림을 전달하는 스크립트
      • 원하는 스크립트를 아무거나 써서 등록할 수 있다!!!!
      • https://github.com/mdn/serviceworker-cookbook/
  2. Indexed DB API
    • 브라우저에서 제공하는 NoSQL DB
    • PWA가 오프라인 기능을 제공할 수 있는 방법 중 하나

PWA의 한계​

  1. 브라우저별 지원 정도가 다름

    • 항상 브라우저가 해당 api 인터페이스를 가지고 있는지 확인하고, 분기 처리 해야함

    • 특히, 애플의 경우 막아놓은거 짱 많음!!!!

      이미지 출처: https://web.dev/articles/http-cache?hl=ko

    • 항상 참인지 확인하고 소스를 가져다 쓰자..

  2. 새로고침 등 방해 요소가 많음
